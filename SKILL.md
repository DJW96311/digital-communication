# 课件 + 作业 → 考试复习材料 (lecture-organizer)

把课程的 **PPT/PDF 课件** 和 **作业题+答案** 两路素材融合，重构成**面向考试**的复习材料。核心理念只有一句话：

> **PPT 定知识结构，作业定考法，题型定输出。**

也就是说：从 PPT 提炼章节主线、知识点、公式；从作业反推老师怎么考；最后按**判断题 / 填空题 / 大题**三种题型组织输出。最终建立一条映射链：

```
PPT 小节 ─▶ 核心知识点 ─▶ 核心公式 ─▶ 对应作业题 ─▶ 判断题 / 填空题 / 大题考法
```

## 三种模式

本 skill 有三种整理粒度，按用户意图选择（**默认走模式一**）：

- **模式一 · 考试复习模式（逐小节，默认）**：以"小节"为单位整理，一节一个 12 部分结构块。**不逐页翻译 PPT**，而是按考试逻辑重构。覆盖作业融合、题型分析、公式速查。用户说"整理这一章""从 x.x 小节开始""结合作业复习""输出判断题/填空题/大题"时走这个模式。
- **模式二 · 逐页深度模式**：以"页"为单位逐页深挖（每页 8 部分框架）。只有当用户**明确**要"逐页""每一页都讲""P1 P2 P3 这样整理"时才走。详见文末"逐页深度模式（备用）"。
- **模式三 · 公式记忆模式（公式族 + 默写，下游）**：以模式一/二产出的 `detail.md` 为输入，把本章公式归并成少数几个**公式族**，产出可打印默写的 PDF（七段结构，含默写空表 + 答案页）。用户说"整理成公式族""公式记忆手册""做成可默写的""标注必背/可推/概念"时走这个模式。详见文末"模式三 · 公式记忆模式"。

下文工作流程默认针对**模式一**。渲染、公式、图表、LaTeX 等通用步骤三模式共用。

## 输入约定（固定文件夹）

课程资料放在固定结构里，直接按约定找：

- **`ppt/`**：课程 PPT 或 PDF 课件。命名可能混乱（`Chapter1.pdf`、`Chapter2.1.pdf`、`Chapter23.pdf`、`Chapter 13 ...pdf`、`chapter 15.pptx` 都可能出现）——见下方"命名歧义"。
- **`homework&answer/`**：作业题、作业答案、习题解析。典型成对出现：`homework0N.pdf` + `Answer0N.pdf`。

输出默认写到课件所在复习目录下、按章分文件夹（如 `<复习目录>/整理部分/第X章/`）。问清或合理推断输出根目录。

## 工作流程（模式一 · 考试复习）

### 第 0 步：扫描两个文件夹，建立"素材清单"

先别急着总结。先把两个文件夹扫一遍，搞清楚**手上有什么**：

1. 列 `ppt/` 下所有文件，初步识别每个文件对应**哪一章/哪一节**（见"命名歧义"）。
2. 列 `homework&answer/` 下所有文件，识别**作业题**和**答案**各有哪些、是否成对。
3. 输出一份"素材清单"给用户看：`Chapter X ← ppt/xxx.pdf ↔ homework&answer/homework0N.pdf + Answer0N.pdf`。

**为什么先列清单**：PPT 命名和作业编号经常对不上（作业 03 不一定是第 3 章），强行匹配会张冠李戴。先让用户确认映射，后面才靠谱。

#### 命名歧义（很重要，最易踩坑）

PPT 文件名风格不统一，**不要凭文件名猜章节归属**。常见坑：

- `Chapter23.pdf`、`Chapter27.pdf` —— 可能是"第 2.3 节 / 第 2.7 节"，也可能是"第 23 章 / 第 27 章"。
- `Chapter41.pdf`、`Chapter43.pdf` —— "第 4.1 / 4.3 / 4.5 节"还是别的？
- `Chapter2.1.pdf` vs `Chapter23.pdf` —— 同一门课两种命名风格并存。

**处理法**：打开文件**看内容**（渲染后看标题页 / 目录页），用**实际标题**判定章节，而不是文件名。文件名只当线索。判定不准就在清单里标"⚠️ 待确认"，列几个可能选项让用户拍板。**宁可问，不要错。**

### 第 1 步：渲染课件为逐页图片（视觉提取是刚需）

课件里大量公式是图片，PDF 文字层常被打散成乱码（如 `|z| = √ a2 + b2`）。**必须用视觉提取**——只读文字会丢关键信息。

用 `scripts/render_slides.py`（PDF 直接渲染；PPTX 先经 PowerPoint COM 转 PDF 再渲染）：

```bash
<VENV_PY> <SKILL_DIR>/scripts/render_slides.py "<课件路径>" --outdir "<渲染目录>" --dpi 150
```

生成 `<渲染目录>/pages/0001.png ...`（逐页图，**主素材**）、`text/0001.txt ...`（文字层，**辅助**，质量参差）、`manifest.json`（页数等元信息）。对照**图片**提取信息，不要信任 text 文件。

> `<VENV_PY>` / `<SKILL_DIR>` 见下方"首次运行：准备 Python 环境"。

### 第 2 步：从 PPT 建立"章节结构"（章 → 小节 → 页码范围）

通读渲染图（分批读，每批 5–8 页，避免上下文爆），建立**本章的结构骨架**：

- 本章**核心主题**是什么（一句话，从标题页和目录页判断）。
- 本章分哪些**小节**（按 PPT 里带编号的标题页切分，如 `2.3 Power Spectral Density`）。
- 每个小节覆盖**哪几页**（记录页码范围，如 §13.1 → P1–P30）。

把骨架记下来（如 `§13.1 Characterization (P1–P30) / §13.2 Signaling (P31–P60) ...`）。这是后面"逐小节整理"和"作业匹配"的索引。

### 第 3 步：扫描作业并匹配到章节（本 skill 的核心增量）

这一步把作业拉进来，是区别于"只看 PPT"的关键。

1. 渲染 `homework&answer/` 里**对应本章**的作业题和答案 PDF（同样用 `render_slides.py` 出图）。
2. **匹配章节**：优先用文件名/内容里的线索（`Chapter 4`、`Ch4`、`HW4`、`Assignment 4`、题干里的主题词）把作业对到 PPT 章节。
3. **匹配不准就停下来问**：如果无法自动判断某份作业属于哪一章，**列出可能匹配项让用户确认，不要强行总结**。例如："homework07 涉及匹配滤波器，可能对应 Chapter 4 或 Chapter 5，请确认。"
4. 匹配好后，把每道作业题粗读一遍，初步判断它考哪个知识点（细节在第 6 步深挖）。

### 第 4 步：判断本章主题，生成本章专属主线

在正式逐小节整理前，先回答：**这一章到底在解决什么问题？** 围绕答案生成本章的**专属主线**——一条把全章知识点串起来的逻辑脉络。

- 主线要**专属**：每章主题不同，主线就不同。"信号如何通过信道"和"如何对抗衰落"是两条完全不同的线。**不要把某一章的主线强行套到其他章节。**
- 主线的作用：后面每个小节整理时，都要说明"本小节在这条主线上的位置"，让全章始终有根线牵着，而不是散落的点。

把主线写一两句记下来，整章总结时会展开成"本章专属主线"和"知识地图"。

### 第 5 步：逐小节考试化整理（12 部分结构，本 skill 的核心产出）

从用户指定的小节开始，**每次只整理一个小节**。整理一个小节时，综合该小节范围内的 PPT 页 + 对应作业题，按下述 **12 部分结构**产出（不必每项都满，有则写、无则跳过，但顺序保持一致）：

1. **本小节共同主题**：一句话点出这节在讲什么。
2. **本小节在本章主线中的位置**：承接什么、为后面什么铺垫（让全章脉络不断）。
3. **逻辑主线**：把这节的概念/公式按因果顺序串成一条线（不是罗列）。
4. **分页重点**：按页码标注本节关键页讲了什么（`P6: 多普勒频移推导`），方便回查 PPT。
5. **核心概念与符号**：本节引入的关键概念（给直观含义，不只定义）+ 符号表（每个符号是什么）。
6. **核心公式与适用条件**：本节的关键公式（LaTeX）。**每个公式必须说明符号、来源、适用条件**——公式不写适用条件等于没写。
7. **必要推导**：关键推导从定义出发**逐步写全，不跳步**，并补"为什么这一步成立"。
8. **对应作业题与考法**：本节对应哪些作业题，老师怎么考这节的知识点。
9. **判断题可能怎么考**：这节的知识点能出什么样的判断题（见"判断题考法重点"）。
10. **填空题可能怎么考**：能出什么样的填空题（见"填空题考法重点"）。
11. **大题答题模板**：这节的知识点出大题时的标准答题步骤/模板。
12. **易错点与考试速记版**：常见混淆 + 一两句话的考前速记。

每整理完一个小节，**停下来等用户说"继续"再进下一节**。这给了用户检查和纠偏的机会，也避免一口气输出过长。

### 第 6 步：作业题反向分析（9 部分结构）

对（本章涉及的）每道作业题，做一次**反向拆解**——不是只给答案，而是拆出"这道题在考什么、怎么考、能怎么变形"：

1. **题目类型**：计算题 / 推导题 / 概念辨析 / 图形分析 …
2. **对应知识点**：对应 PPT 哪个小节、哪个知识点。
3. **关键词触发公式**：题干里哪些词触发哪些公式（建立"看到 X 就用公式 Y"的反射）。
4. **标准解题步骤**：分步写清，可直接搬到考场。
5. **公式来源和适用条件**：用的公式从哪来、什么条件下成立。
6. **答案中的关键推导**：**不跳步**地讲清答案怎么来的，解释为什么这么做。
7. **可变形为判断题 / 填空题 / 大题的方式**：同一考点还能怎么换着法考。
8. **易错点**：这题常见的错法。
9. **考试速记**：一两句话记核心。

**做完 9 部分分析后，再附上「答案原文（Answer）」**：把 `Answer` PDF 里该题的英文解答**逐字贴上**——不翻译、不跳步、保留原文表达与原公式。为什么这么做：分析讲"怎么想"，原文给"标准怎么写"，两者对照既能学解题思路、又能直接背标准英文答题表达。原文用「答案原文」标签单独成盒（转换器会渲染成专门样式的盒子）。

> 用户说"解释这道题为什么这样做，不要跳步"时，重点展开第 6 项，并在「答案原文」里保留 Answer 的每一步。

### 第 7 步：整章考试化总结（13 部分，公式速查 + 一页速查放最前）

当用户要"整章总结"时，输出**考试化总结**——不按 PPT 顺序机械重复，而是提炼成考前能直接用的东西。**结构顺序固定，公式速查和一页速查必须放最前面**（因为考前最先翻的就是这两个）：

1. **公式速查表**：本章所有关键公式一表汇总（公式 + 一句话用途 + 适用条件）。
2. **一页考试速查版**：把全章压缩到一页纸的速查（公式 + 结论 + 易错点），考前一晚翻这一页。
3. **本章知识地图**：知识点之间的结构关系（树/链/框图，用文字描述）。
4. **本章专属主线**：第 4 步那条主线展开。
5. **PPT 小节知识总结**：每小节一句话精华。
6. **作业题考点映射表**：每道作业题 → 对应知识点 → 对应题型。
7. **判断题考点表**：本章可出判断题的考点清单。
8. **填空题考点表**：本章可出填空题的考点清单。
9. **大题模板表**：本章大题的标准答题模板。
10. **高频公式推导链**：考试高频公式的推导链条（互相怎么推出）。
11. **核心概念横向比较表**：易混概念的对比（如 delay spread vs Doppler spread）。
12. **全章易错点**：汇总本章所有易错点。
13. **最终速记版**：最精简的几句话，临考默写用。

### 第 8 步：生成 md → tex → pdf（三种格式，内容一致）

每章输出三种格式（同一份笔记的三个载体）：

1. **`detail.md`**：Markdown 全文（留档、便于搜索编辑）。按「逐小节整理 → 作业题分析 → 整章总结」组织。
2. **`notes.tex`**：用转换器把 `detail.md` 转成 LaTeX，**不要手写**（视觉样式都在转换器里）：
   ```bash
   <VENV_PY> <SKILL_DIR>/scripts/md2tex.py "<detail.md>" "<notes.tex>" --title "<章标题>"
   ```
   转换器自动处理：考试结构标签 → 配色盒子、`**加粗**`/`*斜体*`、`$$公式$$` 居中、表格→tabular（含 `$|a|$` 这种带 `|` 的数学单元格）、Unicode 符号→LaTeX、章节/页眉视觉条。**公式务必用 LaTeX 数学环境**，不要嵌截图。
3. **`notes.pdf`**：编译：
   ```bash
   <VENV_PY> <SKILL_DIR>/scripts/compile_pdf.py "<notes.tex>"
   ```
   跑两遍 xelatex 解交叉引用。编译失败时脚本打印错误上下文——常见是 LaTeX 命令拼错或公式花括号未配对，按提示改 `.tex` 后重编译。

> LaTeX 用 **XeLaTeX + `ctexart`**（中文支持）。**"公式速查"和"一页速查"必须放在文档最前面**——在 `detail.md` 里把"整章总结"的前两项提到逐小节内容之前，或在 `.tex` 里手动调整顺序。

## 三种题型的考法重点

整理判断题/填空题/大题时，按下面的侧重点构思——这是"作业定考法、题型定输出"的落地。

### 判断题（T/F）—— 抓"条件、适用、混淆、特例"

判断题最爱在四个地方设陷阱：

- **条件是否成立**：某个结论需要的条件被偷换/缺失（如"任意信号"其实是"实信号"）。
- **公式是否适用**：某公式被用到不满足适用条件的场景。
- **概念是否混淆**：两个近义概念被互换（如 symbol error rate vs bit error rate）。
- **特殊情况是否被误用**：把特殊情形的结论当成一般结论（如把"等概率下 ML=MAP"当成无条件成立）。

出判断题时：给一句陈述，标出它错在哪一类陷阱上。

### 填空题 —— 抓"定义、公式、符号、结论、取值"

填空题考能精确作答的"硬知识"：

- **定义**：某概念的标准定义。
- **公式**：关键公式的某个部分。
- **符号含义**：公式里某符号代表什么。
- **关键结论**：重要定理/结论。
- **常见取值**：典型参数值、量级。

出填空题时：留一个空，答案唯一且简短。

### 大题 —— 抓"步骤、调用、推导链、图形、模板、表达"

大题考综合应用，答题要可复用：

- **解题步骤**：标准流程（建模 → 选公式 → 代入 → 化简 → 结论）。
- **公式调用**：何时用哪个公式。
- **推导链**：从已知到结论的完整推导。
- **图形分析**：结合星座图/频谱图/波形图的题怎么分析。
- **计算模板**：同类题的固定算法模板。
- **最终表达**：结论用规范英文表达。

出大题模板时：给一套"看到这类题就这么写"的可搬用步骤。

## 必须遵守的原则

1. **不要逐页机械翻译 PPT**。按考试逻辑重构，不是英文翻中文。
2. **先判断章节主题，再生成本章专属主线**。主线专属，不复用。
3. **不要把某一章的主线套到其他章节**。
4. **每次只整理一个小节**，等"继续"再下一节。
5. **公式必须解释符号、来源和适用条件**。没适用条件的公式是半成品。
6. **重要推导不能跳步**，要解释每步为什么成立。
7. **作业题必须反向映射到知识点和考试题型**（这是"作业定考法"的体现）。
8. **对易混概念要反复提醒**（横向比较表 + 易错点 + 速记三处呼应）。
9. **最终输出必须能直接服务考试复习**——每个部分都要想"考前这东西怎么用"。
10. **LaTeX 用 XeLaTeX 可编译版本**，优先 `ctexart`；**"公式速查"和"一页速查"放最前**。

## 通用规则（两模式共用）

### 语言规则

考试常为英文，笔记遵循 **"步骤英文 + 概念中文 + 公式术语英文"**：

- **题干**：保留 PPT / 作业的英文原文，不翻译。
- **推导 / 解题步骤描述**：用**英文**写（如 "Taking the Fourier transform of both sides, we obtain ..."）。步骤可直接搬到英文答题纸。
- **概念含义解释、易错点提醒**：用**中文**写（讲清"为什么、易错在哪"）。
- **公式、专业术语、考试答题表达**：一律**英文**（MAP、sufficient statistic、"the ML detector is equivalent to the minimum-distance detector"）。
- 一句话可中英混排，但**推导步骤本身用英文句子**。

### 公式处理规则

每个公式除转写成 LaTeX，还必须：**逐符号释义**（每个符号是什么）、**只依据 PPT 上下文禁止猜测**（PPT 没说就推断+标注或标待查，绝不瞎猜——猜错的符号比没有更害人）、**给物理含义 + 适用条件**。详见 `references/formula-guide.md`（转写对照表 + 易错点）。

### 图表处理规则

PPT 里的关键图（示意图/波形图/星座图/频谱图）**用文字标注位置，不裁切嵌入**（自动裁切定位易失真）。写一句**图位指引 + 内容描述**：

> 📷 见 PPT p.11：右下方给出基带信号频谱示意图——$|X(f)|$ 集中在零频附近的矩形瓣。

要点：标页码（`见 PPT p.XX`）、用文字提炼图承载的信息、不嵌图不裁切（`crop_figure.py` 仅供用户明确要嵌某图时用）。

### Markdown 写作禁忌（detail.md 里别写什么，很重要）

写 `detail.md` 时**不要用 ASCII 流程图、框线字符（│ ├ ─ ► ▼ ▶ ★）、或 ``` 代码块**来表达结构关系。

为什么：转换器 `md2tex.py` **不把 ``` 代码块当代码逐字处理**——里面的内容会被当成普通正文走转义和数学解析，于是 `e^{-x}` 的 `^`、`½`、`γ_b` 的下划线、框线字符要么直接炸编译（`^`/`{}`/`\@` 在文本或数学模式里非法），要么变成缺字空格。真实踩坑：在知识地图里画 ASCII 树，树里的 `Pe=½e^{-E_b/N₀}` 的 `^` 导致整章编译失败；表格单元里写 `$\gamma_b=... \@10^{-5}$` 的 `\@`（手写"at"）在数学模式里非法。

要表达"结构关系 / 流程 / 层级 / 知识地图"，一律用**项目符号列表**或**表格**代替 ASCII 图（这俩转换器能正确渲染成 `itemize` / `tabular`）。公式永远包在 `$...$` 或 `$$...$$` 里；"at" 写成 `\text{ @ }` 或拆成两个 `$...$` 段，**别写 `\@`**。

### 首次运行：准备 Python 环境

脚本依赖 pymupdf / pywin32 / python-pptx / pillow，**系统 python 没装**。第一次用建专用 venv（之后复用），**用 uv**（全局约定）：

```bash
cd <SKILL_DIR>                                  # 本 SKILL.md 所在目录
uv venv --python 3.14 .venv
uv pip install --python .venv/Scripts/python.exe -r requirements.txt
```

之后所有命令里 `<VENV_PY>` = `<SKILL_DIR>/.venv/Scripts/python.exe`，`<SKILL_DIR>` = 本 SKILL.md 所在目录。**永远用这个 venv 的 python。**

> pywin32 仅 Windows 有（PowerPoint COM 把 PPTX 转 PDF）。macOS/Linux 上 PPTX 输入不可用——先在 PowerPoint 里另存为 PDF 再喂给 skill。

## 视觉设计（PDF 排版规范）

PDF 复习材料的价值靠**视觉层次**。`scripts/md2tex.py` 已把配色固化进转换器——**直接用转换器，无需手调**。考试模式的盒子映射：

| 结构标签 | 视觉处理 | 为什么这么排 |
|---|---|---|
| 核心公式 / 核心公式与适用条件 | **蓝色粗边框"公式卡"，白底居中** | 考试要用的公式 = 视觉主角 |
| 公式速查 / 一页速查 / 速记 | **醒目速查盒** | 考前最先翻 |
| 核心概念 / 核心概念与符号 | 绿色盒子 | 基础知识，温和强调 |
| 必要推导 / 推导过程 | 灰色小字降亮度 | 推导可弱化，快速浏览 |
| 大题答题模板 / 考试写法 | **橙色 callout** | 可直接搬到答题纸 |
| 判断题 / 填空题 / 作业考法 | 各自配色盒（琥珀/青/青绿） | 题型分层，一眼区分 |
| 主题 / 主线 / 前后联系 | 紫色"逻辑链"盒 | 逻辑关系要突出 |
| 易错点 | **红色警示盒** | 高频丢分点，最醒目 |

设计原则：**可用的（公式、大题模板、速查）要跳出来，辅助的（符号释义、推导）要退回去，危险的（易错点）要警告。** 换主题色改 `md2tex.py` 里的 `TITLE_COLOR` 与各盒子定义，全局生效。

## 边界情况

- **PPT/作业章节匹配不上**：列出可能匹配项让用户确认，绝不强行总结。
- **某章没有对应作业**：只做 PPT 驱动的整理，在第 8 项/作业分析处注明"本章无配套作业"。
- **PPTX 打不开/COM 报错**：提示用户在 PowerPoint 里"另存为 PDF"再用 PDF 路径处理。
- **课件是扫描件**：`render_slides.py` 照常出图，视觉提取照常（这正是视觉路径的优势）。
- **一章太长（>60 页）**：严格分批，每批 5–8 页，处理完即写盘暂存。先告诉用户"这章 N 页，我会分 K 批 / 整理成 M 个小节"。
- **用户只要某一部分**（如"只要公式速查""只要大题模板"）：尊重，跳过不需要的部分。

## 逐页深度模式（备用 · 模式二）

仅当用户**明确**要"逐页""每一页都讲"时使用。以页为单位，每页按 **8 部分框架**深挖：

1. **这一页在讲什么** 2. **核心概念** 3. **核心公式** 4. **公式含义/符号释义** 5. **推导过程** 6. **和前面知识的联系** 7. **考试中怎么写** 8. **易错点总结**

逐页 = 每页单独成节、绝不跨页合并、做深度逻辑梳理而非概括。课件相邻几页常是同一主题的逐步展开（一个定理分 3 页推导），合并会丢每页独有细节；逐页能精确对应"这知识点在 PPT 第几页"。完整保留 PPT 原始公式与推导，宁可写长也不为精简丢信息。`md2tex.py` 的 8 部分配色盒对这套标签生效。

> 默认还是走**模式一（逐小节考试化）**。三模式都遵守上面的语言规则、公式规则、图表规则、LaTeX 输出规范。

## 模式三 · 公式记忆模式（公式族 + 默写 PDF）

把一章公式整理成**可打印、可默写**的"公式族"手册。和模式一/二的考试复习笔记是**不同产物**：笔记讲"怎么想、怎么考"，公式记忆手册讲"怎么记、怎么默"。它以**模式一/二产出的 `detail.md` 为输入**（公式已从 PPT 视觉提取并校对），是下游模式。

**核心理念**：不要公式大全，要做**公式族**——一族由一条主公式串起：

```
主公式 ──▶ 变形 ──▶ 特殊情况 ──▶ 常见结论
```

能由主公式现场推出的标 `\ketui`（可推），**不让用户死背**；只有"记不住必丢分"的端点公式标 `\bibei`（必背）。把记忆负担压到最小。

### 输入 / 输出

- **输入**：`lecture-organizer-workspace/<ch>/detail.md`（模式一/二的产物）。公式**逐字采用**，不改写不简化。若不存在，先跑模式一产出它。
- **输出**：`formula-memory-workspace/<ch>/notes.tex → notes.pdf`（独立工作区，与 `lecture-organizer-workspace/` 平级，每章一个子文件夹）。

### 八段结构（固定顺序，全部入册）

一、本章公式记忆总览（主线 + 公式族概览用阿拉伯数字 + 三句口诀）→ 二、公式族卡片（每族一个 `familybox`，含记忆核心/主公式/推导链/常见结论/易混点，公式字段标 `\bibei`/`\ketui`/`\gainian`）→ 三、必背公式极简版（红框）→ **四、Self-test questions（英文）**（`tabularx` 两列，按公式族分组，每族 3–5 题覆盖 F=公式/C=概念/V=变式/D=推导 四类题型，答案列 `\dotfill`）→ **五、Answers**（`\clearpage` 另起页，与四一一对应，答案列 `\af`/`\at`）→ 六、易混公式对照 → **七、Worked problems**（作业 `HW` + PPT 例题 `EX`，每题 `probbox` 含 Problem/Solution/Variant）→ 八、本章公式总账（灰框速记）。

> 第四节提问一律**英文**（公式仍用 LaTeX）；四类题型用 `\qF`/`\qC`/`\qV`/`\qD` 标签。第七节：作业题+规范解答逐字取自 `detail.md` 的「答案原文」，PPT 例题从 `*-render/pages/` 视觉提取，每题配一个变体+参考答案；无配套作业的章节（Ch13/15）只用 `EX`。详见 `references/formula-memory-guide.md`。

### 构建步骤

1. **抄模板**：复制 `templates/formula-memory.tex`（导言区 + 全部宏 `\bibei`/`\ketui`/`\gainian`/`\af`/`\at`/`familybox`/`\core`/`\secbar` 已就绪，本身可编译）到 `formula-memory-workspace/<ch>/notes.tex`，只改正文。**或用脚本半自动起骨架**（自动拷导言区 + 取标题 + 抽候选公式，判断活仍靠人工/代理）：
   ```bash
   <VENV_PY> <SKILL_DIR>/scripts/build_formula_memory.py "<detail.md>" --chapter "<章标题>" --outdir formula-memory-workspace/<ch> --compile
   ```
2. **抽公式族**：通读 `detail.md`，把公式归并成 4–8 个族，每族定一条主公式，按"主公式→变形→特殊情况→结论"串。跨章重复的（如带通表示、信号空间内积）在总账点明"是哪族延伸"，**不重复推导**。
3. **填七段**：每个公式字段挂标注 —— `\bibei`（必背：考试高频、记不住必丢分的端点公式）/ `\ketui`（可推：能由必背现场推出）/ `\gainian`（概念：只需理解含义）。默写表覆盖全部 `\bibei`。
4. **编译自检**：`cd formula-memory-workspace/<ch> && xelatex -interaction=nonstopmode notes.tex`（**跑两遍**）。**必须全过才算完成**：exit 0、日志 `^!` 致命错误 = 0、`no .* in font` 缺字形 = 0、`Overfull` = 0 或极少（长公式用 `align`/`multiline` 折行）。

### 四条排版铁律（违反必编译失败/丢字形，详见 `references/formula-memory-guide.md`）

- **A**. tabularx 单元格里【禁用】`\bm`/`\boldsymbol`（测宽遍崩溃，报 `Improper alphabetic constant`）。答案单元格强调一律用颜色宏 `\af{...}`（数学）/`\at{...}`（文字）。
- **B**. 【禁用】圆圈数字 ①②③④⑤（Latin Modern 无此字形、xeCJK 不路由 → 显示空白）。公式族编号/列表全用阿拉伯数字 1 2 3。
- **C**. 正文里裸 Unicode 箭头 `⇒` 也丢字形，要写 `$\Rightarrow$`。
- **D**. 双参数宏（如 `\inner{x}{y}`）别误用单参数形式，会把后续中文吞进 math 模式而报缺字形。

> **多章批量**：一章一个子代理并行（读 `detail.md` + 抄模板 + 编译自检），并发触发 MiKTeX lock 时 `sleep 6` 重试（最多 3 次）。已用此法铺满全书 9 章实例（见 `formula-memory-workspace/`）。

> **端到端示例（参考样板）**：`formula-memory-workspace/ch2-s2.1/`（带通/低通信号表示，6 页，含默写空表 + 答案页）是完整成品；新建一章时可对照它的 `notes.tex` 结构来填。
